A03 Application convention — monolith structure Major src/App.js · 31,715 lines · 170 funcs
Pattern
  • The whole frontend is one file. There can't be a shared PageHeader primitive when nothing is file-scoped.
  • 170 component declarations in one file (const Btn = …, function Dashboard() { … }, …) — every "page" is just a top-level const.
  • 3,063 <div> elements with inline styles — most of which would otherwise be wrapped components.

Step 7 · suggested execution order tree shape first, then primitives, then mechanical sweeps

  1. Split App.js into ~25 files (A03). Prerequisite — every other axis is easier once components are file-scoped.
  2. Promote in-file primitives (A01): Btn / Badge / Card into ui/, add new PageHeader / Eyebrow / Input / Spinner / NumericCell.
  3. Define token tables in theme.js (A04, A05, A06): CHART_PALETTE, TYPE, RADII — and case-normalise the hex literals scattered through chart configs.
  4. Adoption sweep for the new primitives — 281 raw <button> → <Btn>; 154 raw <input> → <Input>.
  5. Responsive scaffold (A07): one useWindowWidth hook + useCols on top-level grids.
  6. A11y + motion sweep (A08, A09): aria-labels on icon buttons + charts; useReducedMotion gate on the 195 transitions.
  7. Hygiene cherry-on-top (A13). Already clean — keep it that way after the split.
